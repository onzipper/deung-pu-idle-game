# 🎮 ดึ๋งปุ๊ — Game Design Document (GDD)

> **Single source of truth** ของ vision/direction — ถ้าไฟล์อื่น (CLAUDE.md, ROADMAP.md, การ์ดเก่าใน ClickUp) ขัดกับไฟล์นี้ ให้ยึดไฟล์นี้
> **v2 — 2026-07-05**: ปรับใหญ่จาก "ทีม 3 คลาส auto-battler" → **idle MMORPG-lite ตัวละครเดี่ยว** (สรุปจากการคุยกับเจ้าของ vision)
> Roadmap + สถานะงานอยู่ที่ [ROADMAP.md](./ROADMAP.md)

## 🌟 North Star

**"ผู้เล่นปั้นตัวละครของตัวเองจนแข็งพอติดอันดับ Hall of Fame เพื่อขิงกับคนอื่น"**

HOF มีหลายหมวด: เวลสูงสุด · พลังต่อสู้รวม (คำนวณจากโจมตี/ป้องกัน/เลือด ฯลฯ) · แยกตามอาชีพเริ่มต้น
ทุกระบบต้องชี้เข้าเป้านี้ — ฟีเจอร์ไหนไม่พาผู้เล่นเข้าใกล้การติดอันดับ ให้ตั้งคำถามว่าทำไปทำไม

## 🎭 Core Fantasy (เรียงตามที่ผู้เล่นให้ค่า)

1. **Competition** — แข่ง/ขิง (north star)
2. **Collection** — สะสมอาวุธ/ชุด/คลาส
3. **Power progression** — ตัวเลขโต เก่งขึ้นเรื่อย ๆ
4. **Exploration + Social** — สำรวจโลก + เล่นกับเพื่อน (ปาร์ตี้)

## 🧍 ตัวละคร (ฟิล Ragnarok)

- ผู้เล่นสร้างตัวละครได้ **สูงสุด 3 ตัว/บัญชี** (อนาคตอาจขยาย) — เลือก**อาชีพพื้นฐาน 1 สาย**ตอนสร้าง (ดาบ/ธนู/เวท)
- **Power = level + stat พื้นฐานที่แจกแต้มเอง + คลาส/สกิล + gear** — ❌ ไม่มีสายอัปเกรดซื้อด้วยเงินอีก (ระบบ atk/speed/hp เดิมถูกตัด — เงินมีความหมายที่ยา/ของ NPC/ตลาดกลางแทน)
- เวลอัปได้ **แต้ม stat พื้นฐาน**ให้แจกเอง
- **เปลี่ยนคลาสผ่านเควส** เป็นสายต่อ (เช่น ดาบ → อัศวิน → …) แต่ละขั้นมีสกิลชุดใหม่
- **สกิล**: ใช้ทั้ง **cooldown + mana** · ต่อขั้นคลาสมี 2-3 สกิล ปลดตามเวล · ช่อง **auto-cast สูงสุด 3** ปลดตามเวล · สกิลนอกช่อง auto ผู้เล่นกดเอง
- **ตาย → กลับไปเกิดที่เมือง ไม่มีโทษ** (เฟสนี้) · พฤติกรรมหลังตายให้ผู้เล่น**ตั้งค่าเอง** (เช่น auto เดินกลับไปฟาร์มต่อ / รอที่เมือง)

## 🗺️ โลกและการเดิน (ฟิล IdleOn เบา ๆ)

- **Map เป็นธีม** (เช่น map1 = โลกมนุษย์) → แต่ละ map มีหลาย **zone (stage)** ฉากเปลี่ยนไปตาม zone
- ตัวละคร**เดินระหว่าง zone เองได้** (แตะขอบจอ/ปุ่มลูกศร → เดินไป zone ข้างเคียง มีแอนิเมชันเดินจริง) — ไม่ใช่บังคับเดินอิสระ 2D
- Zone ถัดไป**ปลดล็อกด้วยเงื่อนไข** (ฆ่าครบ/เควส) · ปลดแล้ว**ย้อนกลับไปฟาร์ม zone เก่าได้อิสระ** (เลือกฟาร์มตามของดรอป)
- **Fast travel (เคาะ 2026-07-06)**: วาปไป zone ที่ปลดแล้วได้จากหน้าเลือกโซน — ฟรี, มี cast time สั้น, ห้ามมีมอน aggro ติดตัว · **ยันกลับเมือง** = วาปทันทีแม้โดนรุม (ของขายที่ NPC, บอทใช้) · เดินเท้ายังเป็นทางหลักของโซนติดกัน/การปลดโซนใหม่
- ท้าย map เป็น **ห้องบอสพิเศษ** — ต้องดูพิเศษกว่า zone ปกติ ชนะแล้วปลด map ถัดไป
- **Zone = สนามล่ามอน (2026-07-05 แทนระบบเดินหน้าฝ่าเวฟเดิม)**: มอน**สุ่มเกิดกระจายในพื้นที่ zone** (mob cap + respawn timer เป็น config ต่อ zone) ตัวละคร **auto-hunt** วิ่งไล่ตีทีละตัว · มอน 2 นิสัยปนกัน: **passive เป็นหลัก** (ไม่โจมตีก่อน โดนตีแล้วค่อยสู้) + **aggressive มี aggro radius** หนาแน่นขึ้นเมื่อใกล้ห้องบอส · กล้องนิ่งต่อ zone ในแมพแรก ๆ — zone กว้าง + กล้องตามเป็นอัปเดตอนาคต (ออกแบบ config เผื่อ)

## ⚔️ ของดรอปและ Gear

- v1 ดรอปแค่ **อาวุธ + ชุด** จากมอนและบอส (อื่น ๆ ค่อยว่ากัน)
- Gear เป็น **item-instance** (unique ID + ownerId + audit) + **server-authoritative ตั้งแต่ gear ชิ้นแรก** — retrofit ทีหลังแทบเป็นไปไม่ได้ · **item dupe = existential threat**

## 🏘️ เมืองหลัก (hub)

- **NPC shops**: ยาเลือด · ยามานา · ยันกลับเมือง (return scroll) · ของวาปหาสมาชิกปาร์ตี้
- ทางเข้า**ตลาดกลาง** · (อนาคต: กระดานเควส ฯลฯ)

## 💰 ตลาดกลาง (official trading)

- ซื้อขายด้วย**เงินในเกม**ที่ดรอปจากมอน/บอส (ไม่ใช่ RMT-first)
- อนาคตมี **tax/ค่าธรรมเนียม ปรับได้จาก backoffice** โดย admin — **เรื่อง tax ทำท้ายสุด**

## 👥 ปาร์ตี้ (social)

- **สูงสุด 3 คน** (อนาคตขยาย) · เฟสแรก **party-only** (ชวนกันเข้าห้อง) · ปลายทางอยากได้ open-world เห็นคนแปลกหน้าใน map
- **ตอน online**: เห็นตัวละครกันจริง ช่วยกันตี/ใส่สกิล — เทคนิค: **lockstep input-sync** บน deterministic engine + websocket relay (ส่งเฉพาะ input เช่น กดสกิล ไม่ส่ง state ทั้งสนาม) · server ตรวจย้อนหลังด้วย replay
- **แชร์ exp + เงิน** · **ของดรอปจอใครจอมัน**
- **ตอน offline**: สมาชิกกลายเป็น **"ร่างเงา"** — AI ตีต่อด้วยสแตต/สกิล/auto-cast จริงของเจ้าของ แสดงตัวจาง + ป้ายออฟไลน์ · รายได้เข้า pool offline-idle ที่มี cap เดิม (อนาคต: ให้เลือก "ฝากตัวกับปาร์ตี้" หรือ "กลับเมือง")
- มี**ไอเทมวาป**ขายในเมืองสำหรับวาปไปหาสมาชิกปาร์ตี้

## 🎨 Art Direction (สรุปปิด 2026-07-06)

- **คงสไตล์ procedural เดิมของเกม (M4.5-4.6)** — ทดลองใน /proto แล้ว 2 ทาง (pixel-MMX3, ตัวละคร SVG อนิเมะ/RO) เจ้าของไม่ชอบทั้งคู่ → ปิดการทดลอง ลบ /proto (2026-07-06) · การยกเครื่อง art ทั้งเกมพักไว้จนกว่าเจ้าของหยิบขึ้นมาใหม่
- **จุดว้าว = ของสวมใส่ ยังเป็นเป้าหมายหลัก**: paper-doll เห็นชุด/อาวุธจริงบนตัวละคร · อาวุธใหญ่อลังการ · gear tier สูง = ชุดระยิบระยับ + **ออร่าอาวุธลุกโชนแบบซุปเปอร์ไซย่า** — สร้างบน**ริกสไตล์เดิม** (precedent: tier-2 gold accents ของ evolution ทำได้ดีแล้ว) · ผูกกับ M7 gear และเปล่งประกายเต็มที่ตอน M8 party (คนอื่นมองเห็น) · ระวัง blend additive บนฉากสว่าง (บทเรียน /proto: ใช้สีเปลวทึบ+ขอบเข้ม)

## ⚠️ Build Order (จุดที่มักเข้าใจผิดสุด — คงหลักเดิม)

**ลำดับความสำคัญ ≠ ลำดับการสร้าง**: competition เป็น north star แต่**สร้างท้ายสุด** (cold-start ไม่มีคนให้แข่ง · ต้อง server-authoritative + anti-cheat ก่อนอันดับถึงมีความหมาย · live-ops หนัก)

**M5 Character Pivot ⭐ → M6 World & Town → M7 Gear & Drops ⭐ → M8 Party → M9 Economy & Competition**

## 🔑 การตัดสินใจสำคัญ (กระทบ architecture)

1. **Save schema เปลี่ยนเป็น account → characters(≤3) ตั้งแต่ M5**
2. **item-instance + server-authoritative ตั้งแต่ M7** (gear ชิ้นแรก)
3. **Real-time party = lockstep** บน deterministic engine — ต้องมี Node websocket server รันต่อเนื่อง (Hostinger shared hosting ไม่พอ ต้องตัดสินใจ infra/VPS ก่อนเข้า M8)
4. Backoffice (admin ปรับ tax/ค่าธรรมเนียม) — โครงคิดเผื่อตอนทำตลาดกลาง แต่ตัว tax ทำท้ายสุด

## ❌ ตัดออก / ⏸️ พักไว้

- **PvP arena** — ❌ ตัดออกจาก roadmap (ตัดสินใจ 2026-07-05)
- **Prestige / reset loop** — ⏸️ พักไว้ ยังไม่ทำ ตัดสินใจอีกครั้งหลัง M9
